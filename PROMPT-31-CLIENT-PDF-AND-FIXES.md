def generate_scope_summary(self, job_type: str, fields: dict,

Job type: {job_type}

Write 4-8 bullet points describing what will be built and installed.

## PART 2: Concrete Stock Order Fix

### Symptom
Stock order shows: `1 x 20' concrete footing (10.5' needed, 9.5' remaining)`

Concrete footings are individual holes dug and poured on site — not linear stock that comes in sticks. This line makes no sense.

### Fix

In the stock order calculation (find it — likely in `backend/calculators/base.py` or `backend/pricing_engine.py` or `backend/stock_order.py`), skip items where `material_type` is `"concrete"` or profile is `"concrete_footing"`:

```bash
grep -rn "stock.*order\|Stock.*Order\|_build_stock\|stock_list" backend/ --include="*.py" | head -10
```

In whatever function builds the stock order list, add:
```python
# Skip concrete — it's not linear stock
if item.get("profile", "") == "concrete_footing" or item.get("material_type", "") == "concrete":
    continue
```

Instead, if concrete items exist, add a separate line at the bottom of the stock order:
```
Concrete: 7 footings × 12" dia × 42" deep (~2.2 cu yd total)
```

## PART 3: Fix `hss_4x4_0.25` Unrecognized Profile Warning

### Symptom
Every quote shows: `[WARNING] Item 13: Unrecognized profile 'hss_4x4_0.25' in 'Overhead support beam'`

### Root Cause
The profile exists in `PRICE_PER_FOOT` (material_lookup.py) and `STOCK_WEIGHTS` (weights.py), but some other validation is checking a dict where it's missing.

### Fix
```bash
grep -rn "Unrecognized profile\|unrecognized.*profile" backend/ --include="*.py"
```

Find the validation that generates this warning and add `hss_4x4_0.25` to whatever dict it's checking. It's likely a `MATERIAL_TYPES` dict or a `VALID_PROFILES` set that was never updated when we added HSS profiles.

Also add `hss_6x4_0.25` while you're there — same issue would occur if a heavier gate used the larger beam.
